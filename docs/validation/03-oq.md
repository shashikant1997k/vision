# 03 — Operational Qualification (OQ)

Verifies the system operates per requirements. Each case lists a manual procedure
(executed/witnessed by QA) and the **automated tests** that provide supporting
evidence (re-run for regression / re-qualification). Record Pass/Fail + initials.

> Automated evidence: `QT_QPA_PLATFORM=offscreen python -m pytest -q --junitxml=oq-evidence.xml`

## Notes / limits documented for the customer

- **Inline code grade is an approximate "process-control" grade, NOT a certified
  ISO 15415/15416 verifier grade** ([D-012](../decisions/decision-log.md)).
  Certified grading is a sampling activity with a conformant verifier — covered in PQ.
- **Throughput, false-reject rate, and grade correlation** are line-specific and
  verified in **PQ** on the real line at line speed.
- Automated GUI tests run headless (offscreen Qt); on-line PQ confirms the live
  display and operator interactions on the target PC.
